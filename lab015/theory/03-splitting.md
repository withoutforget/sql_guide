# 03 — Разбиение строк на части

[← извлечение и замена](02-extract-trim-replace.md) · [оглавление](../README.md) · [дальше: LIKE / ILIKE / SIMILAR TO →](04-like-similar.md)

---

Обратная к склейке операция — **разбить** строку по разделителю. Три типовые задачи:
«взять N-ю часть», «получить массив частей» и «развернуть части в строки таблицы».
Для каждой — свой инструмент.

## `split_part(строка, разделитель, N)` — взять N-ю часть

Самый практичный инструмент парсинга: разбивает строку по разделителю и возвращает
**одну** часть с номером `N` (**с 1**):

```sql
SELECT split_part('Москва, Тверская, 7', ', ', 1);  -- 'Москва'
SELECT split_part('Москва, Тверская, 7', ', ', 2);  -- 'Тверская'
SELECT split_part('Москва, Тверская, 7', ', ', 3);  -- '7'
```

- Разделитель — **строка** (может быть многосимвольной, здесь `', '` — запятая с
  пробелом), не regex.
- Если части с номером `N` **нет** — возвращается **пустая строка** `''` (не `NULL`,
  не ошибка):

  ```sql
  SELECT split_part('a-b-c', '-', 5);   -- ''   (пятой части нет)
  SELECT split_part('a-b-c', 'x', 1);   -- 'a-b-c'  (разделителя нет → вся строка как 1-я часть)
  ```

- С PostgreSQL 14 `N` может быть **отрицательным** — отсчёт с конца:
  `split_part('a-b-c', '-', -1)` → `'c'` (последняя часть). Удобно, когда важен
  «хвост», а число частей заранее неизвестно.

`split_part` идеально ложится на разбор структурированных строк «город, улица, дом»,
«код-подкод», «домен из `user@домен`» (`split_part(email, '@', 2)`).

## `string_to_array(строка, разделитель)` — в массив

Возвращает **массив** `text[]` из частей. Массивы как тип — большая тема
[lab018](../../lab018/), здесь достаточно знать, что массив можно собрать из строки
и тут же развернуть:

```sql
SELECT string_to_array('a,b,c', ',');   -- {a,b,c}   (значение типа text[])
```

- Второй аргумент `NULL` — разбивает на отдельные **символы**.
- Есть третий необязательный аргумент — какую подстроку считать `NULL`-элементом.

Массив нужен, когда результат пойдёт дальше в SQL: измерить длину
(`array_length`), развернуть (`unnest` из [lab006](../../lab006/)), проверить
вхождение (`'b' = ANY(...)`).

## `string_to_table(строка, разделитель)` — в строки таблицы (PG14+)

Функция, **возвращающая множество** (SRF, см. [lab006](../../lab006/theory/02-unnest-ordinality-srf.md)):
отдаёт по **одной строке результата** на каждую часть. По сути — сразу
`unnest(string_to_array(...))`, но короче:

```sql
SELECT * FROM string_to_table('красный,зелёный,синий', ',') AS t(color);
```
```
  color
---------
 красный
 зелёный
 синий
```

Как SRF, `string_to_table` встаёт во `FROM` наравне с таблицей. Дальше её обычно
**джойнят** к исходной таблице через `LATERAL` (или неявный `LATERAL` для функции),
чтобы развернуть колонку-список по строкам:

```sql
-- развернуть теги каждого товара: одна строка на пару (товар, тег)
SELECT p.name, tag
FROM products AS p,
     LATERAL string_to_table(p.tags, ',') AS tag;
```

А дальше эти строки можно **агрегировать** — например, посчитать, сколько товаров у
каждого тега (`GROUP BY tag`). Это стандартный приём «нормализации» строки-списка в
полноценные строки для аналитики.

> `string_to_table` появилась в PG14. До неё писали
> `unnest(string_to_array(s, sep))` — результат тот же.

## `regexp_split_to_table` / `regexp_split_to_array` — разбиение по regex

Когда разделитель — не фиксированная строка, а **шаблон** (несколько пробелов
подряд, любой знак пунктуации, один из нескольких символов), берут regex-версии:

- `regexp_split_to_table(строка, шаблон [, флаги])` → строки (как `string_to_table`);
- `regexp_split_to_array(строка, шаблон [, флаги])` → массив (как `string_to_array`).

```sql
-- разбить по ЛЮБОМУ числу пробелов подряд (\s+):
SELECT * FROM regexp_split_to_table('one  two   three', '\s+') AS w(word);
```
```
 word
-------
 one
 two
 three
```

Обычный `string_to_table('one  two   three', ' ')` тут дал бы **пустые** элементы
между лишними пробелами — а `\s+` (один или больше пробельных символов) схлопывает
их в один разделитель. Это главная причина брать regex-разбиение: реальные тексты
разделены «грязно».

Синтаксис шаблонов (`\s`, `\d`, классы, квантификаторы) — в
[файле 05](05-posix-regex.md).

## Что выбрать

| Нужно | Инструмент |
|-------|-----------|
| одну конкретную часть (2-ю, последнюю) | `split_part` |
| массив всех частей (пойдёт дальше в SQL) | `string_to_array` / `regexp_split_to_array` |
| развернуть части по строкам (для джойна/агрегации) | `string_to_table` / `regexp_split_to_table` (+ `LATERAL`) |
| разделитель — фиксированная строка | не-regex версии (`split_part`, `string_to_*`) |
| разделитель — шаблон (пробелы, пунктуация) | `regexp_split_to_*` |
