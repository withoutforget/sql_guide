# 02 — Извлечение подстрок, поиск, обрезка, замена

[← базовые функции](01-basic-functions.md) · [оглавление](../README.md) · [дальше: разбиение строк →](03-splitting.md)

---

Здесь — функции, которые **достают** кусок строки, **находят** позицию подстроки,
**срезают** мусор по краям и **заменяют** содержимое. Регулярные выражения
появляются в этом файле только как «мостик» (`substring(... FROM 'regex')`) —
полностью они разобраны в [файлах 05–06](05-posix-regex.md).

## Извлечение подстроки: `substring` и `substr`

### По позиции и длине

```sql
substring(строка FROM старт FOR длина)   -- SQL-стандартный синтаксис
substr(строка, старт, длина)             -- функциональный синтаксис (то же самое)
```

- **Индексация с 1** (не с 0!): первый символ имеет позицию `1`.
- `FOR длина` (или третий аргумент) можно опустить — тогда берётся «до конца строки».

```sql
SELECT substring('привет мир' FROM 1 FOR 6);  -- 'привет'
SELECT substr('привет мир', 8);               -- 'мир'   (с 8-го символа до конца)
SELECT substr('телефон', 1, 3);               -- 'тел'
```

**Поведение на краях** (важно — обычно не роняет запрос, а «подрезает»):

```sql
SELECT substring('abc' FROM 5);      -- ''    (старт за концом → пустая строка, НЕ NULL)
SELECT substr('abc', 0, 2);          -- 'a'   (позиции 0 нет, берётся только 1-я)
SELECT substr('abc', -1, 3);         -- 'a'   (окно -1..1, реальные символы — только 1-й)
SELECT substring(NULL FROM 1);       -- NULL  (вход NULL → NULL)
```

Мысленная модель: `substr` берёт «окно» символов с позиции `старт` длиной `длина` и
оставляет только те позиции окна, что реально существуют в строке (`1..length`).
Отсюда результаты выше: часть окна может «висеть» левее 1 или правее конца — эти
позиции просто отбрасываются.

`left(s, n)` / `right(s, n)` из [файла 01](01-basic-functions.md) — частные удобные
случаи: `left(s, 3)` = `substr(s, 1, 3)`, `right(s, 3)` = «последние 3 символа».

### `substring(строка FROM 'regex')` — извлечение по шаблону

Если во `FROM` стоит **строка-шаблон** (регулярное выражение), `substring` вернёт
**первое совпадение**, а если в шаблоне есть скобочная группа `(...)` — только её:

```sql
SELECT substring('заказ №1990 от' FROM '\d+');       -- '1990'   (первая группа цифр)
SELECT substring('user@mail.ru'  FROM '@(.+)$');     -- 'mail.ru' (то, что в скобках)
SELECT substring('abc' FROM '\d+');                  -- NULL     (совпадения нет)
```

Это самый короткий способ «выдернуть» одну подстроку по шаблону. Полный разбор
синтаксиса regex и родственных функций (`regexp_match`, `regexp_substr`) — в
[файлах 05–06](05-posix-regex.md).

## Поиск позиции подстроки

| Функция | Что возвращает | Пример → результат |
|---------|----------------|--------------------|
| `position(подстрока IN строка)` | позицию (с 1) первого вхождения, `0` если нет | `position('мир' IN 'привет мир')` → `8` |
| `strpos(строка, подстрока)` | то же, но аргументы наоборот | `strpos('привет мир', 'мир')` → `8` |
| `starts_with(строка, префикс)` | `true`/`false` — начинается ли строка с префикса | `starts_with('AB-1234', 'AB')` → `true` |

```sql
SELECT strpos('привет мир', 'zzz');   -- 0   (не найдено → 0, а не NULL)
SELECT position('@' IN 'anna@mail.ru'); -- 5
```

`position`/`strpos` — это буквальный поиск (не по шаблону), возвращающий **число**;
если нужен только факт «есть/нет», удобнее `strpos(...) > 0`, `starts_with(...)`, а
для подстроки — `LIKE '%...%'` (см. [файл 04](04-like-similar.md)). `position`
считает **символы**, не байты (на кириллице позиция логичная, «человеческая»).

## Обрезка краёв: `trim`, `ltrim`, `rtrim`, `btrim`

Убирают **ведущие/хвостовые** символы (по умолчанию — пробелы). Классика — чистка
данных из импорта, где по краям налипли пробелы.

```sql
trim([LEADING | TRAILING | BOTH] [символы] FROM строка)
```

- Без указания стороны — режется с **обеих** сторон (`BOTH`).
- Без указания символов — режутся **пробелы**.
- Можно передать **набор символов** — тогда режется любой символ из набора, пока
  такие идут подряд с края.

```sql
SELECT trim('  привет  ');                 -- 'привет'      (пробелы с обеих сторон)
SELECT trim(LEADING '0' FROM '007');       -- '7'           (ведущие нули)
SELECT trim(BOTH 'x' FROM 'xxabcxx');      -- 'abc'
```

Функциональные варианты — короче для типовых случаев:

| Функция | Эквивалент | Пример → результат |
|---------|-----------|--------------------|
| `ltrim(s [, chars])` | `trim(LEADING ...)` | `ltrim('  hi')` → `'hi'` |
| `rtrim(s [, chars])` | `trim(TRAILING ...)` | `rtrim('hi  ')` → `'hi'` |
| `btrim(s [, chars])` | `trim(BOTH ...)` | `btrim('xxfooxx', 'x')` → `'foo'` |

```sql
SELECT btrim('xxfooxx', 'x');   -- 'foo'   (режет 'x' с обоих концов)
SELECT btrim('  привет  ');     -- 'привет'
```

⚠️ **`trim` режет только КРАЯ, и только заданные символы.** «Двойные пробелы
**внутри** строки» он не тронет: `btrim('  анна   иванова  ')` → `'анна   иванова'`
(внутренние пробелы остались). Чтобы схлопнуть внутренние повторы, нужен
`regexp_replace(s, '\s+', ' ', 'g')` (см. [файл 06](06-regexp-functions.md)) — это
частый второй шаг очистки после `btrim`.

## Замена содержимого: `replace`, `translate`, `overlay`

Три функции с **разной** механикой — их постоянно путают.

### `replace(строка, что, на_что)` — замена подстроки (без regex)

Меняет **все** вхождения подстроки `что` на `на_что`. Это буквальный поиск, без
регулярных выражений:

```sql
SELECT replace('a.b.c', '.', '-');       -- 'a-b-c'   (все точки)
SELECT replace('2024-07-21', '-', '');   -- '20240721'
```

### `translate(строка, из, в)` — посимвольная замена/удаление

`translate` заменяет **каждый символ** из набора `из` на символ на **той же позиции**
в наборе `в`. Это не «подстрока → подстрока», а «символ → символ»:

```sql
SELECT translate('12-34', '-', '');   -- '1234'   (символ '-' удалён)
SELECT translate('a1b2c3', '123', ''); -- 'abc'   (все три цифры удалены)
```

Ключевые правила:
- отображение **по позициям**: 1-й символ `из` → 1-й символ `в`, 2-й → 2-й, и т.д.;
- если символа `из` **нет соответствия** в `в` (набор `в` короче) — символ
  **удаляется**. Пустой `в` = «удалить все перечисленные символы»;
- в отличие от `replace`, за один вызов можно заменить/удалить **несколько разных**
  символов.

```sql
-- транслитерация-lite: каждая кириллическая буква → одна латинская (1:1)
SELECT translate('самовар',
                 'авдезиклмнопрстуц',
                 'avdeziklmnoprstuc');   -- 'samovar'
```

| | `replace` | `translate` |
|--|-----------|-------------|
| единица замены | **подстрока** целиком | **отдельный символ** |
| сколько правил за вызов | одно | много (по позициям наборов) |
| удаление | `replace(s, x, '')` | символ без пары в `в` удаляется |
| regex | нет | нет |

### `overlay(строка PLACING замена FROM позиция [FOR длина])` — вставка/перезапись

`overlay` «накладывает» строку `замена`, начиная с `позиции`, перекрывая `длина`
символов исходной (если `FOR` опущен — перекрывает `length(замена)` символов):

```sql
SELECT overlay('abcdef' PLACING 'XY' FROM 2 FOR 3);   -- 'aXYef'  (перезаписал b,c,d)
SELECT overlay('123456789' PLACING '****' FROM 3 FOR 4); -- '12****789'  (маскирование)
```

Применяют для **маскирования** (спрятать середину номера/карты) и точечной правки по
позиции. Для замены по шаблону, а не по фиксированной позиции, берут
`regexp_replace` ([файл 06](06-regexp-functions.md)).
