# 05 — POSIX-регулярные выражения: синтаксис и операторы

[← LIKE / SIMILAR TO](04-like-similar.md) · [оглавление](../README.md) · [дальше: функции regexp_* и кейсы →](06-regexp-functions.md)

---

Регулярное выражение (regex) — это **язык шаблонов** для описания множества строк:
«две заглавные буквы, дефис, четыре цифры», «что-то, потом `@`, потом домен».
PostgreSQL использует **POSIX ERE** (extended regular expressions) — тот же по духу
синтаксис, что в grep/Perl/Python, с небольшими отличиями. Это самый мощный из
инструментов сопоставления (мощнее `LIKE` и `SIMILAR TO` из [файла 04](04-like-similar.md)).

Этот файл — про **синтаксис** и операторы «совпадает / не совпадает». Функции,
которые regex не просто проверяют, а **извлекают** и **заменяют**
(`regexp_match`, `regexp_replace`, …), — в [файле 06](06-regexp-functions.md).

## Операторы сопоставления `~`

Проверяют, есть ли в строке **подстрока**, подходящая под шаблон (не обязательно вся
строка — если не поставить якоря `^`/`$`). Возвращают `boolean`:

| Оператор | Смысл | Пример → результат |
|----------|-------|--------------------|
| `s ~ p`  | `s` содержит совпадение с `p` (с учётом регистра) | `'AB-1234' ~ '[0-9]{4}'` → `true` |
| `s ~* p` | то же, **без** учёта регистра | `'ноутбук' ~* 'БУК'` → `true` |
| `s !~ p` | НЕ содержит совпадения (регистр важен) | `'abc' !~ '\d'` → `true` |
| `s !~* p`| НЕ содержит (без учёта регистра) | `'ABC' !~* '\d'` → `true` |

```sql
SELECT 'AB-1234' ~ '^[A-Z]{2}-[0-9]{4}$';  -- true   (вся строка = 2 буквы, дефис, 4 цифры)
SELECT 'привет123' ~ '\d';                 -- true   (есть хоть одна цифра)
SELECT 'привет'    ~ '\d';                 -- false  (цифр нет)
```

> Функция **`regexp_like(строка, шаблон [, флаги])`** (PostgreSQL 15+) — то же, что
> `~`, но в виде функции и с флагами отдельным аргументом:
> `regexp_like('AB-12', '^[a-z]{2}', 'i')` → `true` (флаг `i` = без регистра).

С `NULL`: любой из операторов на `NULL`-строке даёт `NULL` (не `true`/`false`).
В `WHERE` это отфильтрует строку; если `NULL` нужно учесть — оберните
`COALESCE(s ~ p, false)`.

## Строим шаблон: кирпичики POSIX ERE

### Якоря — привязка к позиции

| Якорь | Значение |
|-------|----------|
| `^` | начало строки |
| `$` | конец строки |
| `\m` / `\M` | начало / конец **слова** |
| `\y` | граница слова (начало или конец) |

Без якорей шаблон ищется **где угодно** внутри строки. `^...$` требует, чтобы шаблон
описывал **всю** строку целиком — это важно для валидации:

```sql
SELECT 'AB-1234xx' ~ '[A-Z]{2}-[0-9]{4}';    -- true   (совпадение есть ВНУТРИ)
SELECT 'AB-1234xx' ~ '^[A-Z]{2}-[0-9]{4}$';  -- false  (вся строка НЕ такая — есть хвост 'xx')
```

### Классы символов — «один символ из набора»

| Запись | Совпадает с |
|--------|-------------|
| `[abc]` | один из `a`, `b`, `c` |
| `[^abc]` | любой символ, **кроме** `a`, `b`, `c` |
| `[a-z]` | диапазон: любая строчная латинская буква |
| `[A-Za-z0-9]` | буква или цифра |
| `[а-яё]` | строчная кириллическая буква (диапазон) |
| `.` | **любой** символ (кроме перевода строки) |

Внутри `[...]` большинство метасимволов теряют силу и означают себя: `[.-]` — это
точка или дефис (дефис в конце/начале класса — литерал).

### Сокращённые классы (шорткаты)

| Шорткат | Эквивалент | Значение |
|---------|-----------|----------|
| `\d` | `[[:digit:]]` | цифра |
| `\D` | `[^[:digit:]]`| не цифра |
| `\w` | `[[:alnum:]_]` | буква, цифра или `_` |
| `\W` | не «словесный» символ | |
| `\s` | `[[:space:]]` | пробельный (пробел, таб, перевод строки) |
| `\S` | не пробельный | |

> **Экранирование в PostgreSQL.** В обычном строковом литерале Postgres (при
> стандартной настройке `standard_conforming_strings = on`, а она включена по
> умолчанию) обратный слэш передаётся в regex **как есть**, поэтому `'\d'` пишется
> просто `'\d'` — без удвоения. Если увидите предупреждение о `\`, можно писать
> POSIX-классы (`[[:digit:]]`) — они без слэша и всегда однозначны.

### POSIX-классы `[[:...:]]` — то же словами

Работают **только внутри** `[...]`. Полезны, когда важна локаль (кириллица тоже
считается буквой):

| Класс | Значение |
|-------|----------|
| `[[:digit:]]` | цифра |
| `[[:alpha:]]` | буква (в т.ч. кириллица — по локали) |
| `[[:alnum:]]` | буква или цифра |
| `[[:space:]]` | пробельный символ |
| `[[:upper:]]` / `[[:lower:]]` | заглавная / строчная буква |
| `[[:punct:]]` | знак пунктуации |

```sql
SELECT 'Тест'  ~ '^[[:alpha:]]+$';   -- true   (кириллица — тоже буквы)
SELECT 'ЁЖ'    ~ '^[[:upper:]]+$';   -- true
```

### Квантификаторы — «сколько раз»

| Кв. | Повторов | Пример |
|-----|----------|--------|
| `*` | 0 или больше | `a*` — `''`, `'a'`, `'aaa'` |
| `+` | 1 или больше | `\d+` — «одна или больше цифр» |
| `?` | 0 или 1 (необязательный) | `colou?r` — `color`/`colour` |
| `{n}` | ровно `n` | `\d{4}` — ровно 4 цифры |
| `{n,}` | `n` или больше | `\d{2,}` |
| `{n,m}` | от `n` до `m` | `\d{2,4}` |

### ⚠️ Жадность и ленивость

По умолчанию квантификаторы **жадные** — захватывают как можно **больше**. Добавив
`?` после квантификатора (`*?`, `+?`, `??`, `{n,m}?`), делаем его **ленивым** —
захватывает как можно **меньше**:

```sql
SELECT substring('<a><b>' FROM '<.*>');    -- '<a><b>'  (жадный .* сожрал всё до последнего '>')
SELECT substring('<a><b>' FROM '<.*?>');   -- '<a>'     (ленивый .*? остановился на первом '>')
```

Это классический источник багов при разборе тегов/скобок: жадный `.*` «перепрыгивает»
через нужную границу. Хотите «до ближайшего» — берите ленивый `.*?` или уточняйте
класс (`[^>]*` — «сколько угодно чего угодно, кроме `>`»).

### Группировка `( )`, альтернатива `|`, обратные ссылки `\1`

- `(...)` — **группа**: объединяет часть шаблона (для квантификатора или чтобы её
  потом **извлечь** — см. файл 06). `(?:...)` — группа **без захвата** (только
  группирует, не запоминается).
- `|` — **или**: `кот|пёс` совпадёт с `кот` либо `пёс`. Внутри группы:
  `(buy|sell)`.
- `\1`, `\2`, … — **обратная ссылка** на текст, пойманный N-й группой (внутри того
  же шаблона), и `\1` в строке замены `regexp_replace` (файл 06):

```sql
SELECT 'тест тест' ~ '(\w+) \1';   -- true   (\1 = повтор того же слова)
SELECT 'тест мир'  ~ '(\w+) \1';   -- false  (слова разные)
```

## Флаги

Модифицируют разбор. Передаются либо третьим аргументом функций
(`regexp_replace(s, p, r, 'gi')`), либо встроенно в начале шаблона `(?i)`:

| Флаг | Действие |
|------|----------|
| `i` | без учёта регистра (как `~*`) |
| `g` | **global** — обрабатывать **все** совпадения, а не только первое (для `regexp_replace`, `regexp_matches`; для булевых операторов смысла не имеет) |
| `m` | многострочный: `^`/`$` совпадают у границ каждой строки, а не всего текста |
| `x` | «расширенный»: пробелы в шаблоне игнорируются, можно комментировать — для читаемости длинных regex |

```sql
SELECT regexp_replace('a1b2', '\d', '#', 'g');  -- 'a#b#'   (флаг g — все цифры)
SELECT regexp_replace('a1b2', '\d', '#');       -- 'a#b2'   (без g — только первую)
```

## Компактный справочник метасимволов

| Символ(ы) | Значение | Пример шаблона | Совпадёт |
|-----------|----------|----------------|----------|
| `.` | любой символ | `a.c` | `abc`, `a7c` |
| `^` `$` | начало / конец строки | `^abc$` | ровно `abc` |
| `\d` `\D` | цифра / не цифра | `\d+` | `1990` |
| `\w` `\W` | словесный / нет | `\w+` | `user_1` |
| `\s` `\S` | пробел / не пробел | `\s+` | `'   '` |
| `[...]` `[^...]` | класс / отрицание | `[A-Z]` `[^0-9]` | буква / не цифра |
| `*` `+` `?` | 0+, 1+, 0/1 | `ab*` | `a`, `ab`, `abb` |
| `{n}` `{n,m}` | точный / диапазон повторов | `\d{2,4}` | `12`, `1234` |
| `*?` `+?` | ленивые версии | `<.*?>` | `<a>` |
| `(...)` | группа (захватываемая) | `(\d+)` | ловит цифры в группу 1 |
| `(?:...)` | группа без захвата | `(?:ab)+` | `abab` |
| `\|` | или | `cat\|dog` | `cat` или `dog` |
| `\1` | обратная ссылка на группу 1 | `(.)\1` | `aa`, `оо` |
| `\m` `\M` `\y` | начало / конец / граница слова | `\mкот\M` | слово `кот` целиком |
| `[[:alpha:]]` | POSIX-класс (буква и т.п.) | `[[:alpha:]]+` | `Тест` |
