# 06 — Функции `regexp_*`: извлечение, замена, практические кейсы

[← синтаксис POSIX regex](05-posix-regex.md) · [оглавление](../README.md) · [дальше: README лабы →](../README.md)

---

Операторы `~` из [файла 05](05-posix-regex.md) отвечают лишь «да/нет». Настоящая сила
регулярок — **извлекать** и **заменять** по шаблону. Это делают функции `regexp_*`.
Часть из них появилась в PostgreSQL 15; у нас PG16 — доступны все.

## `regexp_replace(строка, шаблон, замена [, флаги])` — замена по шаблону

Самая используемая regex-функция. Заменяет совпадения шаблона на строку `замена`:

```sql
SELECT regexp_replace('a-b-c', '-', '_');        -- 'a_b-c'   (без 'g' — только ПЕРВОЕ совпадение!)
SELECT regexp_replace('a-b-c', '-', '_', 'g');   -- 'a_b_c'   (флаг 'g' — все)
```

⚠️ **Без флага `g` заменяется только первое совпадение.** Для «заменить всё» почти
всегда нужен `'g'` — забыть его легко.

В строке `замена` доступны **обратные ссылки** `\1`, `\2`, … на скобочные группы
шаблона — это позволяет **переставлять** и переформатировать куски:

```sql
-- 'ГГГГ-ММ-ДД' → 'ДД.ММ.ГГГГ'
SELECT regexp_replace('2024-07-21', '(\d+)-(\d+)-(\d+)', '\3.\2.\1');  -- '21.07.2024'
```

Два кейса-рабочих лошадки:

```sql
-- оставить только цифры (нормализация телефона): всё «не-цифра» \D → ничего
SELECT regexp_replace('+7 (999) 123-45-67', '\D', '', 'g');   -- '79991234567'

-- схлопнуть повторяющиеся пробелы в один (чистка после trim)
SELECT regexp_replace('Иван   Петров', '\s+', ' ', 'g');      -- 'Иван Петров'
```

## Извлечение: `regexp_match`, `regexp_matches`, `regexp_substr`

### `regexp_match(строка, шаблон [, флаги])` → массив групп первого совпадения

Возвращает **массив** `text[]` с захваченными группами первого совпадения, либо
`NULL`, если совпадения нет. Если групп `()` в шаблоне нет — в массиве один элемент:
всё совпадение.

```sql
SELECT regexp_match('amount=1990 rub', '(\w+)=(\d+)');  -- {amount,1990}   (группы 1 и 2)
SELECT regexp_match('нет цифр', '\d+');                 -- NULL            (нет совпадения)
SELECT (regexp_match('user@mail.ru', '@(.+)$'))[1];     -- 'mail.ru'       (первая группа)
```

`regexp_match` **не** размножает строки: одна входная строка → одно значение
(массив или `NULL`). Индекс `[1]`, `[2]` достаёт нужную группу. Часто удобнее
`substring(s FROM 'шаблон')` из [файла 02](02-extract-trim-replace.md) — он сразу
вернёт **текст** первой группы (а не массив).

### `regexp_matches(строка, шаблон [, флаги])` → набор строк

Функция, **возвращающая множество** (SRF): по строке-результату на **каждое**
совпадение. Обычно с флагом `'g'` — «найти все».

⚠️ **Подводный камень: `regexp_matches` — это SRF.** Поставленная в список `SELECT`,
она **размножает строки** запроса (одна входная строка с 3 совпадениями превратится
в 3 выходных):

```sql
-- ОПАСНО в SELECT: одна строка входа → несколько строк выхода
SELECT 'x', regexp_matches('a1b2', '\d', 'g');
--  x | 1
--  x | 2       ← строка «раздвоилась»
```

Правильное место `regexp_matches('...', 'g')` — во **`FROM`** (обычно через
`LATERAL`), тогда размножение строк осознанно и контролируемо:

```sql
-- извлечь ВСЕ числа из текста и развернуть по строкам
SELECT (m)[1] AS num
FROM regexp_matches('цена 199 и 800 руб', '\d+', 'g') AS m;
--  199
--  800
```

Каждый результат — по-прежнему **массив групп** (потому `(m)[1]`). Без `'g'`
`regexp_matches` вернёт максимум одну строку — тогда проще взять `regexp_match`
(единственное число) и не связываться с SRF.

### `regexp_substr(строка, шаблон [, старт [, N [, флаги [, группа]]]])` — N-е совпадение (PG15+)

Возвращает **текст** совпадения (а не массив) — часто удобнее `regexp_match`:

```sql
SELECT regexp_substr('a1b2c3', '\d');            -- '1'   (первое совпадение)
SELECT regexp_substr('a1b2c3', '\d', 1, 2);      -- '2'   (2-е совпадение, старт с 1-й позиции)
SELECT regexp_substr('foo=42', '\w+=(\d+)', 1, 1, '', 1);  -- '42'  (группа 1 первого совпадения)
```

## Считаем и ищем позицию: `regexp_count`, `regexp_instr` (PG15+)

```sql
SELECT regexp_count('a1b2c3', '\d');    -- 3   (сколько совпадений)
SELECT regexp_instr('abc123', '\d');    -- 4   (позиция первого совпадения, с 1; 0 если нет)
```

`regexp_count` — быстрый способ «сколько цифр/слов/тегов в строке», без разворота в
SRF.

## Сводка функций

| Функция | Возвращает | `NULL`/нет совпадения | Появилась |
|---------|-----------|-----------------------|-----------|
| `regexp_replace(s,p,r[,f])` | строку с заменами | без совпадений — исходная `s` | давно |
| `regexp_match(s,p[,f])` | `text[]` групп первого совпадения | `NULL` | PG10 |
| `regexp_matches(s,p[,f])` | **набор** `text[]` (SRF!) | 0 строк | давно |
| `regexp_substr(s,p,...)` | текст совпадения | `NULL` | PG15 |
| `regexp_count(s,p,...)` | число совпадений | `0` | PG15 |
| `regexp_instr(s,p,...)` | позицию совпадения | `0` | PG15 |
| `regexp_like(s,p[,f])` | `boolean` (как `~`) | — | PG15 |
| `substring(s FROM 'p')` | текст первой группы | `NULL` | давно |

## Практические кейсы (собраны в задачах лабы)

**Нормализация телефона** к `+7XXXXXXXXXX`. Убираем всё, кроме цифр, и берём
последние 10 значащих (одинаково для `+7…`, `8…`, `7…`):

```sql
SELECT '+7' || right(regexp_replace('8 (916) 000-11-22', '\D', '', 'g'), 10);
--  '+79160001122'
```

**Извлечение домена** из e-mail — тремя способами (выбор по вкусу):

```sql
SELECT substring('user@mail.ru' FROM '@(.+)$');  -- 'mail.ru'  (группа через substring)
SELECT split_part('user@mail.ru', '@', 2);       -- 'mail.ru'  (проще, без regex)
SELECT (regexp_match('user@mail.ru', '@(.+)$'))[1];
```

**Валидация e-mail** — простым шаблоном. Полностью «правильно» валидировать e-mail
регуляркой **невозможно** (стандарт RFC чудовищно сложен), но отсеять явный мусор —
легко и полезно:

```sql
SELECT 'anna@mail.ru' ~ '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$';  -- true
SELECT 'bad-email@'   ~ '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$';  -- false
```

Читается так: «непустой локальный кусок, `@`, домен, точка, зона из ≥2 букв, и всё
это — вся строка (`^...$`)». Для `NULL` оберните в `COALESCE(... , false)`.

**Разбор строки `key=value`** — извлечь все пары или конкретные поля:

```sql
-- все пары ключ=значение из строки (в LATERAL — осознанное размножение строк)
SELECT kv[1] AS key, kv[2] AS val
FROM regexp_matches('user=anna action=buy amount=1990', '(\w+)=(\w+)', 'g') AS kv;

-- достать конкретное поле:
SELECT substring('user=anna action=buy amount=1990' FROM 'amount=(\d+)')::int;  -- 1990
```

## Где заканчивается эта тема

- **Полнотекстовый поиск** (`to_tsvector`, `@@`, ранжирование) и **нечёткий /
  триграммный** поиск (`pg_trgm`, `similarity`) — для «умного» поиска по словам с
  учётом морфологии и опечаток — это [lab016](../../lab016/). Regex ищет по точному
  шаблону; полнотекст — по смыслу слов.
- **JSON-строки** (`->`, `jsonb`, `json_agg`) — [lab017](../../lab017/).
- **Массивы как тип** (операторы, функции массивов сверх `unnest`/`string_to_array`)
  — [lab018](../../lab018/).
- **Регистронезависимое сравнение** «по-взрослому»: тип `citext` (сравнивает без
  учёта регистра прозрачно) и явные `COLLATE`-правила. Здесь мы обходились
  `lower(...)`/`ILIKE`.
- **`to_char(число, формат)`** для форматирования чисел (разделители разрядов) —
  относится к functions-formatting и датам ([lab014](../../lab014/theory/05-formatting.md));
  для строк почти не нужен.
