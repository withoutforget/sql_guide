# 05 — Изменение `jsonb`: `||`, `jsonb_set`, `jsonb_insert`, `-`, `#-`, `strip_nulls`

[← содержание и jsonpath](04-containment-jsonpath.md) · [оглавление](../README.md) · [дальше: разворачивание и агрегация →](06-unfold-aggregate.md)

JSON-документ иногда надо **изменить**: добавить атрибут, поправить значение,
выкинуть ключ. Важно понять сразу: все функции ниже **не меняют** исходный
документ, а **возвращают новый** `jsonb` (значения в SQL неизменяемы). Поэтому в
этой лабе мы применяем их в **выражениях `SELECT`** — видим результат, но саму
таблицу не трогаем. Реальный `UPDATE ... SET attributes = jsonb_set(...)` —
тема lab019; приёмы построения нового значения при этом ровно те же.

Все операции — для `jsonb`.

## `||` — слияние/конкатенация

Оператор `||` **сливает** два `jsonb`. Поведение зависит от того, что слева и
справа.

### Объект + объект: добавить/заменить ключи

```sql
SELECT attributes || '{"warranty":24,"color":"чёрный"}'::jsonb
FROM products WHERE id = 10;
```
```
 {"brand": "Logitech", "color": "чёрный", "warranty": 24, "wireless": true}
```

Ключи справа **добавляются**; при совпадении ключа **правое значение
перекрывает** левое (у «Мыши» было `warranty: null`, стало `24`).

### Массив + массив: конкатенация

```sql
SELECT '[1,2]'::jsonb || '[3,4]'::jsonb;    -- [1, 2, 3, 4]
```

Добавить один элемент в массив: `arr || '["x"]'::jsonb` (справа — массив из
одного элемента). А `'[1,2]'::jsonb || '3'::jsonb` даёт `[1, 2, 3]` (скаляр
приклеивается как элемент).

### Подводный камень: `||` для объектов — ПОВЕРХНОСТНОЕ слияние

`||` сливает только **верхний уровень**. Если ключ есть с обеих сторон и его
значение — вложенный объект, правый объект **заменит левый целиком**, а не
сольётся вглубь:

```sql
SELECT '{"a":{"x":1,"y":2}}'::jsonb || '{"a":{"z":3}}'::jsonb;
-- {"a": {"z": 3}}      ← НЕ {"a":{"x":1,"y":2,"z":3}}! Вложенный объект заменён.
```

Чтобы изменить **вложенное** значение, не затирая соседей, нужен `jsonb_set` с
путём (ниже).

## `jsonb_set(target, path, new_value [, create_if_missing])`

Точечно **заменяет** значение по **пути** `text[]`. Аргументы:
- `target` — исходный документ;
- `path` — путь `'{a,b}'` до изменяемого места (как в `#>`);
- `new_value` — новое значение (**`jsonb`**! число пишем как `'4'::jsonb` или
  `to_jsonb(4)`);
- `create_if_missing` (необязательный, по умолчанию `true`) — создавать ли ключ,
  если его нет. `false` → если пути нет, документ не меняется.

```sql
SELECT
    jsonb_set(attributes, '{ram}', '4', true)   AS set_or_create,  -- ram → 4 (создан бы, если б не было)
    jsonb_set(attributes, '{ram}', '4', false)  AS set_only        -- меняет, только если ram уже есть
FROM products WHERE id = 10;   -- у «Мыши» ram НЕТ
```
```
             set_or_create             |            set_only
---------------------------------------+---------------------------------
 {"ram": 4, "brand": "Logitech", ...}  | {"brand": "Logitech", ...}   ← не изменён
```

Изменение **вложенного** поля (то, что `||` делает неправильно):

```sql
SELECT jsonb_set('{"a":{"x":1,"y":2}}', '{a,x}', '99');
-- {"a": {"x": 99, "y": 2}}     ← сосед y на месте
```

Путь может указывать и на **элемент массива** по индексу: `jsonb_set(items,
'{0,qty}', '5')` — поле `qty` нулевой позиции.

## `jsonb_insert(target, path, new_value [, insert_after])`

Похож на `jsonb_set`, но **вставляет**, а не заменяет — прежде всего для
**массивов**: добавить элемент по индексу, не затирая существующий.

```sql
SELECT
    jsonb_insert('[1,2,3]', '{1}', '9')          AS before,  -- перед индексом 1
    jsonb_insert('[1,2,3]', '{1}', '9', true)    AS after;   -- после индекса 1
```
```
   before    |    after
-------------+-------------
 [1, 9, 2, 3] | [1, 2, 9, 3]
```

`insert_after` (по умолчанию `false`) — вставлять до или после указанной позиции.
Для объекта `jsonb_insert` добавит ключ, только если его **ещё нет** (иначе
ошибка — в отличие от `jsonb_set`, который молча заменяет).

## Удаление: `-` и `#-`

| Оператор | Что удаляет |
|----------|-------------|
| `doc - 'key'` | ключ верхнего уровня (объект) |
| `doc - int` | элемент массива по индексу (можно отрицательный) |
| `doc - text[]` | сразу несколько ключей верхнего уровня |
| `doc #- '{path}'` | элемент по **пути** (в т.ч. вложенный / элемент массива) |

```sql
SELECT
    attributes - 'wireless'          AS drop_key,    -- убрать ключ
    attributes - array['ram','ssd']  AS drop_many    -- убрать несколько
FROM products WHERE id = 2;
```
```sql
-- #- удаляет по пути: первый цвет смартфона (colors[0])
SELECT (attributes #- '{colors,0}') -> 'colors'   -- ["blue"]
FROM products WHERE id = 1;
```
```
 ["blue"]
```

`-` с числом на массиве: `'[10,20,30]'::jsonb - 0` → `[20, 30]`; `... - -1` →
`[10, 20]` (последний).

## `jsonb_strip_nulls(doc)` — убрать ключи с JSON null

Рекурсивно удаляет из объектов **пары, где значение — JSON null** (напомним из
файла [01](01-json-vs-jsonb.md): JSON null — заданное значение, не «ключа нет»).
Удобно чистить документ перед сохранением/отдачей.

```sql
SELECT jsonb_strip_nulls(attributes)
FROM products WHERE id = 10;   -- было "warranty": null
```
```
 {"brand": "Logitech", "wireless": true}     ← warranty удалён
```

Важно: `jsonb_strip_nulls` трогает **только** JSON null-значения в объектах;
элементы массива, равные null, он **не** убирает (`'[1,null,2]'` останется как
есть) — чтобы не сдвигать индексы.

## Собираем вместе (пример из задачи 11)

```sql
SELECT
    name,
    attributes                                        AS before,
    attributes || '{"warranty":24,"color":"чёрный"}'  AS after_merge,
    jsonb_set(attributes, '{ram}', '4', true)         AS after_set,
    attributes - 'wireless'                           AS after_drop,
    jsonb_strip_nulls(attributes)                     AS after_strip
FROM products WHERE id = 10;
```

Все четыре колонки — **независимые** новые документы, построенные из одного и того
же `attributes`; сам `attributes` в таблице не изменился.

## Итог файла

- Все операции возвращают **новый** `jsonb`; таблицу не меняют (мутации — lab019).
- `||` — слияние (объекты: правый перекрывает; **поверхностно!**) / конкатенация
  массивов.
- `jsonb_set(t, path, val, create_if_missing)` — заменить/создать по пути;
  `new_value` — это `jsonb`. `jsonb_insert` — вставить (особенно в массив).
- Удаление: `- 'key'` / `- idx` / `- text[]` / `#- '{path}'`.
- `jsonb_strip_nulls` — выкинуть ключи со значением JSON null (в объектах).
