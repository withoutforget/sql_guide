# SQL Guide — от нуля до профессионала

Практический тренажёр по SQL: от базового `SELECT` до оконных функций, сложных
джойнов, CTE, рекурсии и приёмов, которые нужны аналитикам и инженерам данных
в реальной работе.

Каждая тема — это **лаба** в отдельной подпапке (`lab001/`, `lab002/`, ...).
Внутри лабы:

- `README.md` — теория, описание доменной области и постановка задач.
- `init.sql` — схема и наполнение тестовыми данными.
- `queries/` — эталонные SQL-запросы, по одному файлу на задачу.
- `run.sh` — автоматический прогон: поднимает БД, накатывает схему, выполняет
  все запросы и печатает результат.

Все манипуляции проводятся на одном экземпляре **PostgreSQL**, который
поднимается через Docker Compose из корня репозитория.

📚 Подборка курсов, тренажёров и документации на русском — в
[`RESOURCES.md`](RESOURCES.md).

## Требования

- Docker и Docker Compose (`docker compose version` должно работать)
- Bash

## Быстрый старт

```bash
# 1. Поднять PostgreSQL (один раз на все лабы)
docker compose up -d

# 2. Запустить любую лабу
cd lab001
./run.sh
```

`run.sh` идемпотентен: пересоздаёт схему лабы с нуля при каждом запуске, поэтому
запускать можно сколько угодно раз.

Остановить и удалить БД со всеми данными:

```bash
docker compose down -v
```

## Интерактивный режим (psql shell)

Чтобы быстро зайти в БД и писать запросы вручную — есть скрипт-обёртка:

```bash
./psql.sh            # открыть psql
./psql.sh lab001     # открыть psql сразу в схеме lab001
                     # (можно писать `SELECT * FROM products` без префикса)
```

Скрипт сам поднимет контейнер, если он не запущен. Полезные команды внутри:

| Команда       | Что делает                                  |
|---------------|---------------------------------------------|
| `\dn`         | список схем                                 |
| `\dt lab001.*`| таблицы схемы lab001                        |
| `\d products` | структура таблицы                           |
| `\x`          | вертикальный вывод для широких строк         |
| `\timing on`  | показывать время выполнения запросов         |
| `\e`          | открыть последний запрос в редакторе         |
| `\q`          | выйти                                        |

Либо напрямую, без обёртки:

```bash
docker compose exec postgres psql -U sql -d sql_guide
```
