# 06 — `NULL` и функции обработки: `COALESCE`, `NULLIF`, `GREATEST`, `LEAST`

## `NULL` есть у любого типа

`NULL` — это не «ноль» и не «пустая строка», а **отсутствие значения**
(«неизвестно / не задано»). `NULL` может быть у столбца **любого** типа: числа,
строки, булева, даты. Ключевое свойство: **любая операция с `NULL` даёт `NULL`**
(кроме специальных проверок `IS NULL`):

```sql
SELECT 100 - NULL;         -- NULL
SELECT NULL || 'текст';    -- NULL
SELECT NULL = NULL;        -- NULL   (не true! сравнивать через IS NULL)
```

Это причина коварных багов: одна `NULL`-скидка в `revenue - discount` обнуляет
(точнее — «онуляет», делает `NULL`) всю строку, а `sum()` потом молча её
выкидывает. Три значения логики (`TRUE`/`FALSE`/`NULL`) подробно разобраны в
[lab001, файл 02](../../lab001/theory/02-where.md). Ниже — четыре функции, которые
приручают `NULL`. Все они приводят свои аргументы к **общему типу** (см. файл 05).

## `COALESCE(a, b, c, ...)` — первый не-`NULL`

Возвращает **первый аргумент, который не `NULL`**; если все `NULL` — вернёт
`NULL`. Классика для «значения по умолчанию»:

```sql
SELECT coalesce(NULL, NULL, 42, 7);   -- 42
SELECT coalesce(discount_rub, 0);     -- NULL-скидку превращаем в 0 для арифметики
SELECT revenue_rub - coalesce(discount_rub, 0);  -- корректная разность, без «заражения» NULL
```

Многоуровневый вариант — «взять первый доступный источник»:

```sql
SELECT coalesce(nickname, full_name, 'аноним');  -- ник → имя → заглушка
```

Тип результата — общий тип аргументов. Поэтому `coalesce(discount_rub, 'нет')`
потребует, чтобы оба были совместимы: `coalesce(discount_rub::text, 'нет')`.

## `NULLIF(a, b)` — `NULL`, если равны

Возвращает `NULL`, если `a = b`, иначе — `a`. Ровно **два** аргумента. Главное
применение — **защита от деления на ноль**: превращаем опасный `0` в `NULL`, и
деление даёт `NULL` (пусто) вместо ошибки:

```sql
SELECT nullif(5, 5);    -- NULL
SELECT nullif(5, 0);    -- 5
SELECT orders_cnt::numeric / NULLIF(visits, 0);  -- если visits=0 → NULL, а не «division by zero»
```

Без `NULLIF` строка с `visits = 0` уронила бы весь запрос ошибкой деления на
ноль (см. задачу 3). Обе функции — `COALESCE` и `NULLIF` — уже мелькали в
прошлых лабах ([lab001](../../lab001/theory/04-case.md), воронки в
[lab012](../../lab012/)); здесь важно понимать их как инструмент работы с типами
и `NULL`.

## `GREATEST(...)` / `LEAST(...)` — максимум/минимум ИЗ АРГУМЕНТОВ

Важно не спутать с агрегатами `MAX`/`MIN`:

- `MAX`/`MIN` — **агрегаты**: берут одно значение из **многих строк** (по вертикали);
- `GREATEST`/`LEAST` — **функции**: берут самое большое/малое из **своих
  аргументов** в **одной строке** (по горизонтали).

```sql
SELECT greatest(3, 9, 1, 7);   -- 9
SELECT least(3, 9, 1, 7);      -- 1
```

### Сюрприз: `GREATEST`/`LEAST` **игнорируют `NULL`**

В отличие от обычной арифметики (где `NULL` «заражает» результат), эти функции
**пропускают** `NULL`-аргументы и работают по остальным. `NULL` вернётся, только
если **все** аргументы `NULL`:

```sql
SELECT greatest(3, NULL, 7, 1);   -- 7    (NULL пропущен)
SELECT least(3, NULL, 7, 1);      -- 1
SELECT least(1290, NULL);         -- 1290 (! не NULL — второй аргумент проигнорирован)
SELECT greatest(NULL, NULL);      -- NULL (все аргументы NULL)
```

Это удивляет: `least(revenue, discount)` при `discount IS NULL` вернёт `revenue`,
а не `NULL`. Поведение отличается и от агрегатов (те тоже игнорируют `NULL`, но
это отдельное правило), и от простого сравнения. Демонстрация — в задаче 10.

Практика — **«пол» и «потолок» значения** в одной строке:

```sql
SELECT greatest(bonus, 0);          -- не даём уйти в минус (пол = 0)
SELECT least(discount, max_allowed); -- не превысить лимит (потолок)
SELECT greatest(net, 1000);          -- сумма к оплате не ниже 1000 ₽
```

### Сводка четырёх функций

| Функция              | Аргументов | Что возвращает                         | `NULL`-поведение |
|----------------------|-----------|----------------------------------------|------------------|
| `COALESCE(a,b,…)`    | любое     | первый не-`NULL`                       | `NULL`, если все `NULL` |
| `NULLIF(a, b)`       | ровно 2   | `NULL`, если `a = b`; иначе `a`        | — |
| `GREATEST(a,b,…)`    | любое     | самый большой из аргументов            | **игнорирует** `NULL` |
| `LEAST(a,b,…)`       | любое     | самый малый из аргументов              | **игнорирует** `NULL` |
