# 03 — `ROLLUP` и `CUBE`: готовые наборы итогов

[← GROUPING SETS](02-grouping-sets.md) · [оглавление](../README.md) · [дальше: GROUPING() и NULL →](04-grouping-function-nulls.md)

---

`GROUPING SETS` требует перечислять наборы руками. Но два случая встречаются
настолько часто, что для них есть отдельные сокращения: **иерархические итоги**
(`ROLLUP`) и **все возможные комбинации** (`CUBE`). Оба — просто короткая запись
для определённого `GROUPING SETS`.

## `ROLLUP` — иерархические подытоги («сворачивание справа налево»)

`ROLLUP (a, b, c)` строит **иерархию** уровней, последовательно **отбрасывая
колонки справа**:

```
ROLLUP (a, b, c)  =  GROUPING SETS (
    (a, b, c),   -- самый детальный уровень
    (a, b),      -- убрали c  → подытог
    (a),         -- убрали b  → подытог
    ()           -- убрали a  → общий итог
)
```

То есть `n` колонок дают **`n + 1` набор**. Мысленная модель — «сворачивание
справа налево» (roll up): начинаем с полной детализации и по одной сворачиваем
колонки с правого конца, каждый раз получая всё более крупный подытог, пока не
свернём всё до общего итога.

### Пример: `ROLLUP (category, subcategory)`

Наша естественная иерархия — **категория → подкатегория**. Хотим: детализацию по
подкатегориям, подытог по каждой категории и общий итог:

```sql
SELECT category, subcategory, SUM(amount) AS revenue
FROM sales
GROUP BY ROLLUP (category, subcategory)
ORDER BY category NULLS LAST, subcategory NULLS LAST;
```
```
  category   | subcategory |  revenue
-------------+-------------+------------
 Дом         | Кухня       |  150000.00
 Дом         | Освещение   |  100000.00
 Дом         | ∅           |  250000.00   ← подытог по «Дом» (subcategory свёрнута)
 Книги       | Детская     |   70000.00
 Книги       | Учебная     |   80000.00
 Книги       | ∅           |  150000.00   ← подытог по «Книги»
 Электроника | Наушники    |  100000.00
 Электроника | Смартфоны   |  500000.00
 Электроника | ∅           |  600000.00   ← подытог по «Электроника»
 ∅           | ∅           | 1000000.00   ← общий итог (обе свёрнуты)
(10 rows)
```

Разбор — три уровня в одном результате:

- набор `(category, subcategory)` → 6 детальных строк;
- набор `(category)` → 3 строки-подытога (в них `subcategory = ∅`);
- набор `()` → 1 строка общего итога (обе колонки `∅`).

6 + 3 + 1 = **10 строк**. И суммы сходятся: `Кухня 150k + Освещение 100k = 250k`
подытог по «Дом»; сумма трёх подытогов `250 + 150 + 600 = 1000k` — общий итог.

> **`ROLLUP` не переставляет колонки.** `ROLLUP (a, b)` и `ROLLUP (b, a)` — разные
> иерархии: первая даёт подытоги по `a`, вторая — по `b`. Порядок колонок = порядок
> сворачивания, ставь колонки от «крупной» к «мелкой» (категория, потом
> подкатегория; год, потом месяц, потом день).

### Когда использовать `ROLLUP`

Когда между колонками есть **естественная иерархия** «от общего к частному» и
нужны подытоги на каждом уровне вложенности:

| Иерархия | `ROLLUP (...)` |
|----------|----------------|
| время | `ROLLUP (year, month, day)` — итоги за год / месяц / день |
| география | `ROLLUP (country, region, city)` |
| каталог | `ROLLUP (category, subcategory, product)` |
| оргструктура | `ROLLUP (department, team, employee)` |

Признак «мне нужен `ROLLUP`»: подытоги нужны **только по префиксам** списка (по
`a`, по `a,b`, по `a,b,c`), но **не** по одному лишь `b` в отрыве от `a` («месяц
без года» смысла не имеет). Если нужны и «одинокие» разрезы — это уже `CUBE` или
явный `GROUPING SETS`.

## `CUBE` — все комбинации измерений

`CUBE (a, b)` строит группировки по **всем подмножествам** перечисленных колонок:

```
CUBE (a, b)  =  GROUPING SETS (
    (a, b),   -- оба измерения
    (a),      -- только a
    (b),      -- только b
    ()        -- ни одного (общий итог)
)
```

`n` колонок дают **`2ⁿ` наборов** (все подмножества): `CUBE(a,b)` → 4,
`CUBE(a,b,c)` → 8, и так далее. Это «полный куб»: итоги по каждому измерению
по отдельности, по каждой их комбинации и общий.

### Пример: `CUBE (category, channel)`

Два **независимых** измерения — категория и канал. Хотим все срезы:

```sql
SELECT category, channel, SUM(amount) AS revenue
FROM sales
GROUP BY CUBE (category, channel)
ORDER BY category NULLS LAST, channel NULLS LAST;
```
```
  category   | channel |  revenue
-------------+---------+------------
 Дом         | offline |  120000.00
 Дом         | online  |  130000.00
 Дом         | ∅       |  250000.00   ← итог по «Дом» (по обоим каналам)
 Книги       | offline |   40000.00
 Книги       | online  |  110000.00
 Книги       | ∅       |  150000.00   ← итог по «Книги»
 Электроника | offline |  240000.00
 Электроника | online  |  360000.00
 Электроника | ∅       |  600000.00   ← итог по «Электроника»
 ∅           | offline |  400000.00   ← итог по offline (по всем категориям)
 ∅           | online  |  600000.00   ← итог по online
 ∅           | ∅       | 1000000.00   ← общий итог
(12 rows)
```

Здесь есть то, чего `ROLLUP (category, channel)` **не дал бы**: строки
`∅ | offline | 400k` и `∅ | online | 600k` — итоги по каналу **в отрыве** от
категории. `ROLLUP` считает подытоги только по префиксам (`category`,
`category+channel`), а `CUBE` добавляет ещё и «одинокий» `channel`. Именно этим
`CUBE` отличается от `ROLLUP`: 4 набора против 3.

### Когда использовать `CUBE`

Когда измерения **независимы** (между ними нет отношения «часть–целое») и нужны
итоги по **любой** их комбинации — например, для интерактивного отчёта/сводной
таблицы, где пользователь крутит разрезы как хочет: «покажи по категориям», «по
каналам», «по категориям и каналам сразу». Осторожно с числом колонок: `2ⁿ`
растёт быстро (`CUBE` из 5 колонок = 32 набора).

## Как `ROLLUP`/`CUBE` раскрываются в `GROUPING SETS`

И `ROLLUP`, и `CUBE` — это буквально сокращённая запись `GROUPING SETS`. Проверим
на `ROLLUP (category, subcategory)`: распишем его как явный `GROUPING SETS` и
сравним с примером выше.

```sql
SELECT category, subcategory, SUM(amount) AS revenue
FROM sales
GROUP BY GROUPING SETS ((category, subcategory), (category), ())
ORDER BY category NULLS LAST, subcategory NULLS LAST;
```
```
  category   | subcategory |  revenue
-------------+-------------+------------
 Дом         | Кухня       |  150000.00
 Дом         | Освещение   |  100000.00
 Дом         | ∅           |  250000.00
 Книги       | Детская     |   70000.00
 Книги       | Учебная     |   80000.00
 Книги       | ∅           |  150000.00
 Электроника | Наушники    |  100000.00
 Электроника | Смартфоны   |  500000.00
 Электроника | ∅           |  600000.00
 ∅           | ∅           | 1000000.00
(10 rows)
```

Результат **идентичен** `ROLLUP (category, subcategory)` — потому что это одно и
то же. Сводная таблица раскрытий:

| Сокращение | Эквивалент `GROUPING SETS` | Число наборов |
|------------|----------------------------|---------------|
| `ROLLUP (a)` | `((a), ())` | 2 |
| `ROLLUP (a, b)` | `((a,b), (a), ())` | 3 |
| `ROLLUP (a, b, c)` | `((a,b,c), (a,b), (a), ())` | 4 |
| `CUBE (a, b)` | `((a,b), (a), (b), ())` | 4 |
| `CUBE (a, b, c)` | `((a,b,c),(a,b),(a,c),(b,c),(a),(b),(c),())` | 8 |

Держи в голове это соответствие — оно снимает почти всю «магию»: любой `ROLLUP` и
`CUBE` можно мысленно развернуть в список наборов и сразу понять, какие строки
будут в ответе.

## Комбинирование в одном `GROUP BY`

В `GROUP BY` можно смешивать обычные колонки, `GROUPING SETS`, `ROLLUP` и `CUBE`
через запятую. Правило одно, но важное:

> Несколько элементов в `GROUP BY` **перемножаются** — итоговый список наборов
> есть **декартово произведение** наборов каждого элемента.

- **Обычная колонка + `ROLLUP`.** `GROUP BY category, ROLLUP (channel)` = обычная
  колонка `category` (всегда в группировке) × наборы `ROLLUP(channel)` =
  `{(channel), ()}`. Итог: наборы `(category, channel)` и `(category)`. То есть
  «внутри каждой категории — детализация по каналу плюс подытог по категории», но
  **без** общего итога по всей таблице (категория из группировки не убирается):

```sql
SELECT category, channel, SUM(amount) AS revenue
FROM sales
GROUP BY category, ROLLUP (channel)
ORDER BY category, channel NULLS LAST;
```
```
  category   | channel |  revenue
-------------+---------+-----------
 Дом         | offline | 120000.00
 Дом         | online  | 130000.00
 Дом         | ∅       | 250000.00   ← подытог по «Дом», общего итога НЕТ
 Книги       | offline |  40000.00
 Книги       | online  | 110000.00
 Книги       | ∅       | 150000.00
 Электроника | offline | 240000.00
 Электроника | online  | 360000.00
 Электроника | ∅       | 600000.00
(9 rows)
```

- **Два `ROLLUP`/`CUBE` рядом** перемножаются и дают много наборов:
  `GROUP BY ROLLUP(a), ROLLUP(b)` = `{(a),()} × {(b),()}` =
  `{(a,b),(a),(b),()}` — то же, что `CUBE(a,b)`. С этим легко нечаянно взорвать
  число строк, поэтому комбинируй осознанно.

## `ROLLUP` vs `CUBE` vs `GROUPING SETS` — что выбрать

| Нужно | Бери |
|-------|------|
| Подытоги по иерархии (префиксам списка колонок) + общий итог | `ROLLUP` |
| Итоги по всем комбинациям независимых измерений | `CUBE` |
| Произвольный, нестандартный список наборов (не префиксы и не полный куб) | `GROUPING SETS` |
| Один уровень группировки | обычный `GROUP BY` |

`ROLLUP` и `CUBE` — это удобство для двух частых форм. Всё, что они умеют, умеет и
`GROUPING SETS`; но там, где подходит `ROLLUP`/`CUBE`, они короче и яснее выражают
намерение.

## Что дальше

Во всех примерах выше строки-итоги помечены `∅` (`NULL`) в свёрнутых колонках. Но
у нас в данных `region` **сам по себе** бывает `NULL` («не указан»). Как отличить
«итоговый `NULL`» от «настоящего `NULL`»? Для этого есть функция `GROUPING()` — ей
посвящён [файл 04](04-grouping-function-nulls.md).
