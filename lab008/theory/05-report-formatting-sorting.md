Расширенная группировка даёт **правильные числа**, но «сырой» вывод с `∅` в
итогах читать неудобно. Этот файл — про то, как превратить его в аккуратный
отчёт: понятные подписи и итоги, стоящие там, где их ждёт глаз.

## Проблема сортировки: куда денутся итоги?

Строки-итоги содержат `NULL` в свёрнутых колонках. А `NULL` при сортировке ведёт
себя особо: по умолчанию в `ORDER BY ... ASC` он уходит **в конец** (`NULLS
LAST`), в `DESC` — **в начало** (`NULLS FIRST`). Значит, если просто написать
`ORDER BY category`, строка общего итога (`category IS NULL`) улетит в конец или
начало **случайно относительно смысла**, а подытоги перемешаются с деталями
непредсказуемо.

Нам же обычно нужна строгая структура:

```
деталь
деталь
  подытог по группе
деталь
деталь
  подытог по группе
    ОБЩИЙ ИТОГ
```

Управлять этим «на глаз» через `NULLS FIRST/LAST` по колонке-измерению — хрупко.
Правильный инструмент — сортировать по `GROUPING(...)`.

## `ORDER BY` с `GROUPING()` — итоги под своей группой

Идея: `GROUPING(X)` = `0` для деталей и `1` для итога по `X`. Если поставить
`GROUPING(X)` в `ORDER BY`, детали (`0`) окажутся **выше** своего итога (`1`) —
ровно как в бумажном отчёте.

Разложим сортировку `ROLLUP (category, subcategory)` по шагам:

```sql
SELECT category, subcategory, SUM(amount) AS revenue
FROM sales
GROUP BY ROLLUP (category, subcategory)
ORDER BY
    GROUPING(category),      -- 1) общий итог (GROUPING(category)=1) — в самый низ
    category,                -- 2) категории по алфавиту
    GROUPING(subcategory),   -- 3) внутри категории: детали (0) выше подытога (1)
    subcategory;             -- 4) детали по алфавиту
```
```
  category   | subcategory |  revenue
-------------+-------------+------------
 Дом         | Кухня       |  150000.00
 Дом         | Освещение   |  100000.00
 Дом         | ∅           |  250000.00
 Книги       | Детская     |   70000.00
 Книги       | Учебная     |   80000.00
 Книги       | ∅           |  150000.00
 Электроника | Наушники    |  100000.00
 Электроника | Смартфоны   |  500000.00
 Электроника | ∅           |  600000.00
 ∅           | ∅           | 1000000.00
(10 rows)
```

Читаем `ORDER BY` как приоритеты сверху вниз:

1. `GROUPING(category)` — все строки, где категория **не** свёрнута (`0`), идут
   раньше строки общего итога (`1`). Итог гарантированно **внизу**.
2. `category` — блоки категорий по алфавиту.
3. `GROUPING(subcategory)` — **внутри** категории детали (`0`) идут выше подытога
   (`1`), поэтому подытог «Дом ∅» встаёт **под** своими «Кухня»/«Освещение», а не
   над ними.
4. `subcategory` — детали внутри блока по алфавиту.

Такой шаблон сортировки — `GROUPING(старшая), старшая, GROUPING(младшая),
младшая, ...` — универсален для иерархических отчётов: он всегда ставит каждый
подытог сразу под его деталями, а общий итог — в самый низ. (В примерах прошлых
файлов мы для простоты писали `ORDER BY ... NULLS LAST` — это работает на нашей
конкретной сортировке, но `GROUPING()` надёжнее и выразительнее.)

## Многоуровневые подписи

Соберём «человеческий» отчёт: вместо `∅` — понятные подписи на каждом уровне.
Комбинируем `CASE` + `GROUPING()` (из [файла 04](04-grouping-function-nulls.md))
для обеих колонок:

```sql
SELECT
    CASE WHEN GROUPING(category) = 1 THEN 'ИТОГО'
         ELSE category END          AS категория,
    CASE WHEN GROUPING(subcategory) = 1 AND GROUPING(category) = 0
              THEN '  · итого по категории'
         WHEN GROUPING(subcategory) = 1 THEN ''       -- в строке общего итога подкатегорию не подписываем
         ELSE subcategory END       AS подкатегория,
    SUM(amount) AS revenue
FROM sales
GROUP BY ROLLUP (category, subcategory)
ORDER BY category NULLS LAST, GROUPING(subcategory), subcategory;
```
```
  категория  |      подкатегория      |  revenue
-------------+------------------------+------------
 Дом         | Кухня                  |  150000.00
 Дом         | Освещение              |  100000.00
 Дом         |   · итого по категории |  250000.00
 Книги       | Детская                |   70000.00
 Книги       | Учебная                |   80000.00
 Книги       |   · итого по категории |  150000.00
 Электроника | Наушники               |  100000.00
 Электроника | Смартфоны              |  500000.00
 Электроника |   · итого по категории |  600000.00
 ИТОГО       |                        | 1000000.00
(10 rows)
```

Что здесь сделано:

- `GROUPING(category) = 1` → строка общего итога, подписываем «ИТОГО»;
- в колонке подкатегории две ветки итога: подытог **категории**
  (`GROUPING(subcategory)=1 AND GROUPING(category)=0`) подписан «· итого по
  категории», а в строке **общего** итога подкатегорию оставляем пустой — иначе
  подпись задвоилась бы;
- отступ (`  ·`) визуально показывает вложенность подытога под деталями.

Обрати внимание: сортировать по подписи-`CASE` **нельзя** (она станет
`'ИТОГО' < 'Дом'` по алфавиту и всё развалится) — поэтому в `ORDER BY` сортируем
по **исходным** колонкам и `GROUPING()`, а не по красивым меткам.

## Форматирование чисел

Меры удобно оформлять уже знакомыми средствами из lab001: `ROUND()` для
округления, деление на 1000 для «в тысячах», конкатенация с единицами измерения.
Например, `ROUND(SUM(amount) / 1000, 1) || ' тыс.₽'`. Это не специфика
расширенной группировки — работает так же, как в обычном `GROUP BY`.

## Чек-лист оформления отчёта с итогами

1. **Подписи** — через `CASE WHEN GROUPING(col)=1 ...`, ветку `GROUPING`
   ставь раньше ветки про настоящий `NULL`.
2. **Сортировка** — по исходным колонкам и `GROUPING()`, а **не** по
   `CASE`-подписям. Шаблон: `GROUPING(старш.), старш., GROUPING(младш.), младш.`
3. **Итог — внизу** своей группы: этого добивается `GROUPING(col)` в начале
   соответствующего уровня `ORDER BY`.
4. Числа — `ROUND()` и прочее оформление, как в обычных запросах.
