# lab009 — Статистические и сортирующие/гипотетические агрегаты, коллекции

Девятая лаба продолжает **модуль C — «Агрегация и аналитика»**. В
[lab001](../lab001/) мы считали `COUNT`/`SUM`/`AVG`, в [lab008](../lab008/) —
управляли тем, **по каким разрезам** считать итоги (`GROUPING SETS`/`ROLLUP`/
`CUBE`). Теперь разберём, **что именно** считать внутри группы, когда обычных
агрегатов мало.

Три семейства «продвинутых» агрегатов:

- **коллекции** — собрать значения группы в список/строку/массив
  (`string_agg`, `array_agg`);
- **статистические** — описать группу числами: разброс (`stddev`, `variance`) и
  связь двух мер (`corr`, `regr_*`);
- **сортирующие и гипотетические** — работать с порядком: медиана и перцентили
  (`percentile_cont`/`_disc`), самое частое значение (`mode`), «какое место занял
  бы» (`rank() WITHIN GROUP`).

Главная сквозная мысль лабы: **одним средним арифметическим легко соврать** — для
честного описания данных нужны медиана, перцентили и стандартное отклонение.

Теория разбита на короткие документы в папке [`theory/`](theory/) — читайте по
порядку. Общая подборка курсов и тренажёров — в [`../RESOURCES.md`](../RESOURCES.md).

## Доменная область

Тот же интернет-магазин. Данные подобраны так, чтобы **демонстрировать** каждую
конструкцию (все числа проверены прогоном — см. комментарии в
[`init.sql`](init.sql)). Четыре таблицы:

### `products` — товары (главный источник числовых мер)
| Колонка | Тип | Смысл |
|---------|-----|-------|
| `id` | INT (PK) | идентификатор |
| `name` | TEXT | название |
| `category` | TEXT | `Электроника`, `Книги`, `Дом`, `Игры` |
| `price` | NUMERIC | цена, ₽ — для медианы, перцентилей, стандартного отклонения |
| `units_sold` | INT | продано штук — для корреляции цена ↔ спрос |
| `rating` | INT | оценка 1..5 — для `mode` (самого частого значения) |

### `customers` / `orders` / `order_items`
Клиенты, шапки заказов и позиции заказов (`order_id`, `product_id`, `qty`). Из
`order_items` собираются «составы заказов» через `string_agg`/`array_agg`.

> **Как подобраны данные (главное):**
> - **Перекос:** в `Электронике` цены `3000…90000` — ноутбук за `90 000`
>   раздувает **среднее** (`21 500`), а **медиана** остаётся типичной (`8 000`).
> - **`cont` ≠ `disc`:** при чётном числе товаров `percentile_cont(0.5)` ≠
>   `percentile_disc(0.5)` (`Электроника`: `8000` ≠ `7000`; `Книги`: `800` ≠
>   `700`); при нечётном — совпадают (`Дом`, `Игры`).
> - **`mode`:** среди рейтингов чаще всего `5` (10 раз против 8 и 2).
> - **Корреляция:** дороже → меньше продаж; сильнее всего в `Доме` (`corr ≈
>   −0.93`), в `Книгах` связи почти нет (`+0.05`).
> - **Списки:** заказы дают осмысленные `string_agg`/`array_agg` (состав заказа,
>   категории клиента).

> **Про хардкор.** Задача 13 — цельная «карточка распределения» на каждую
> категорию: два центра (среднее и медиана), квартили одним `percentile_cont(ARRAY
> [...])` с раскладкой по индексам, стандартное отклонение и — главное — **разрыв
> `среднее − медиана` как числовой признак перекоса**, переведённый в словесный
> вердикт через `CASE` относительно IQR. Всё это без единой оконной функции.
> Изюм — увидеть, что «Электроника» с виду дороже всех, но её среднее — иллюзия
> одного ноутбука. Задача 14 — найти категорию с максимальной **по модулю**
> корреляцией: обычный `corr()` по группам, отсортированный по `abs(corr(...))`
> (оконных ранго́в нельзя — это lab010), с ярлыком силы и подтверждением через `R²`
> регрессии. Обе решаются ровно материалом lab001–lab009.

## Что дальше

Все агрегаты этой лабы **схлопывают** группу в одну строку. В `lab010` —
**оконные функции** (`OVER`, `PARTITION BY`, ранги): те же идеи разбиения, долей и
ранго́в, но **без схлопывания** — результат считается рядом с **каждой** строкой,
а детальные записи сохраняются. В частности, там появятся «настоящие» оконные
`rank()`/`dense_rank()` — те, что дают ранг каждой строке (не путать с
гипотетическими `rank() WITHIN GROUP` из этой лабы).
