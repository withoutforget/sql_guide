# 01 — Функции, возвращающие множества, и `generate_series`

---

Во всех прошлых лабах источником строк во `FROM` была **таблица** (или результат
`JOIN`, подзапроса, `UNION`). В этой лабе появляется ещё один источник —
**функция, которая возвращает не одно значение, а целый набор строк**. Начнём с
понятия и с главной такой функции — `generate_series`.

## Что такое функция, возвращающая множество (SRF)

Обычная функция возвращает **одно значение** на один вызов: `length('abc')` → `3`,
`upper('abc')` → `'ABC'`, `now()` → одна временная метка. Такие функции называют
**скалярными**.

**Функция, возвращающая множество** (по-английски *set-returning function*, SRF) —
это функция, которая на один вызов отдаёт **набор строк** (ноль, одну или много).
Её результат — не число, а маленькая «таблица», и поэтому её естественное место —
в `FROM`, там же, где стоят обычные таблицы.

Мысленная модель простая: **SRF — это генератор строк**. Мы описываем правило
(«числа от 1 до 10», «все даты июня», «элементы этого массива»), а функция
разворачивает его в строки, которые дальше обрабатываются обычным `SELECT` —
их можно фильтровать `WHERE`, соединять `JOIN`, группировать `GROUP BY`.

Самые ходовые SRF в PostgreSQL:

| Функция | Что порождает |
|---------|---------------|
| `generate_series(...)` | ряд чисел или ряд дат/времени с заданным шагом |
| `unnest(массив)` | по строке на каждый элемент массива |
| `string_to_table(строка, разделитель)` | разбивает строку на строки-подстроки |
| `regexp_split_to_table(строка, шаблон)` | то же, но разделитель — регулярное выражение |

`unnest` и строковые SRF разберём в [файле 02](02-unnest-ordinality-srf.md), а
здесь сосредоточимся на самой универсальной — `generate_series`.

## `generate_series` — числовой ряд

`generate_series(start, stop [, step])` порождает **по строке на каждое число** от
`start` до `stop` **включительно**, с шагом `step`.

```sql
SELECT n
FROM generate_series(1, 5) AS g(n);
```
```
 n
---
 1
 2
 3
 4
 5
(5 rows)
```

`g(n)` — это алиас: `g` именует сам источник, а `n` — его единственную колонку.
Дальше `n` — обычная колонка, с ней можно делать что угодно:

```sql
SELECT n, n * n AS square
FROM generate_series(1, 10) AS g(n);   -- числа и их квадраты (это задача 1)
```

### Аргументы и их смысл

- **`start`** — с какого числа начать (включается в результат).
- **`stop`** — до какого числа идти (включается, если шаг «дотягивает» до него).
- **`step`** — шаг; **по умолчанию `1`**, если не указан. Может быть больше
  единицы и **отрицательным** (тогда ряд убывает).

```sql
SELECT array_agg(n) FROM generate_series(1, 10, 2) AS g(n);
-- {1,3,5,7,9}   -- шаг 2: нечётные числа

SELECT array_agg(n) FROM generate_series(10, 2, -2) AS g(n);
-- {10,8,6,4,2}  -- отрицательный шаг: ряд идёт вниз
```

### Поведение в пограничных случаях (важно знать)

- **Пустой результат — не ошибка.** Если до `stop` нельзя дойти в направлении
  шага, функция возвращает **ноль строк**:
  ```sql
  SELECT count(*) FROM generate_series(5, 1) AS g(n);   -- шаг +1, а надо вниз → 0
  ```
- **`NULL` в аргументе → ноль строк** (тоже без ошибки):
  ```sql
  SELECT count(*) FROM generate_series(1, NULL) AS g(n);   -- 0
  ```
- **Шаг `0` — ошибка** (иначе ряд был бы бесконечным):
  ```sql
  SELECT * FROM generate_series(1, 5, 0);
  -- ERROR:  step size cannot equal zero
  ```

Кроме `int`, `generate_series` работает и с `bigint`, и с `numeric` (тогда шаг
может быть дробным, например `0.5`).

## `generate_series` — ряд дат и времени

Та же функция умеет порождать **ряд дат/времени**. Форма —
`generate_series(start, stop, step)`, где `start`/`stop` — метки времени, а `step`
— **интервал** (`interval '1 day'`, `'1 month'`, `'1 hour'` и т. п.). Шаг здесь
обязателен (для времени «по умолчанию 1» смысла не имеет).

```sql
SELECT d
FROM generate_series(
        date '2024-06-01',
        date '2024-06-05',
        interval '1 day') AS g(d);
```
```
           d
------------------------
 2024-06-01 00:00:00+00
 2024-06-02 00:00:00+00
 2024-06-03 00:00:00+00
 2024-06-04 00:00:00+00
 2024-06-05 00:00:00+00
(5 rows)
```

Обратите внимание: хотя мы передали `date`, функция возвращает **метку времени**
(`timestamp with time zone`) — с нулевым временем `00:00:00`. Чаще всего в отчёте
нужна именно дата, поэтому результат **приводят к `date`**:

```sql
SELECT d::date AS day
FROM generate_series(date '2024-06-01', date '2024-06-05', interval '1 day') AS g(d);
```
```
    day
------------
 2024-06-01
 2024-06-02
 2024-06-03
 2024-06-04
 2024-06-05
(5 rows)
```

Шаг-интервал бывает любым — это и делает функцию удобной для отчётов по периодам:

```sql
-- первые числа месяцев: июнь, июль, август, сентябрь
SELECT d::date
FROM generate_series(date '2024-06-01', date '2024-09-01', interval '1 month') AS g(d);
-- 2024-06-01, 2024-07-01, 2024-08-01, 2024-09-01

-- каждые 6 часов в пределах суток
SELECT d
FROM generate_series(timestamp '2024-06-01 00:00',
                     timestamp '2024-06-02 00:00',
                     interval '6 hours') AS g(d);
-- 00:00, 06:00, 12:00, 18:00, 00:00 следующего дня
```

**Зачем это нужно.** Сплошной ряд дат — это «каркас календаря» без пропусков. Если
просто сгруппировать заказы по дате, дни, в которые заказов не было, в отчёт **не
попадут** — появятся дыры. А если сначала построить `generate_series` всех дней, а
затем приклеить к нему факты через `LEFT JOIN`, пустые дни сохранятся с нулями.
Этот приём — **gap-filling** — разберём подробно в [файле 05](05-cases-topn-gapfill.md),
а применим в задачах 11, 12 и 14.

## SRF в `FROM`: полноценный источник строк

Раз SRF возвращает набор строк, она встаёт в `FROM` наравне с таблицами — и с ней
работает весь знакомый арсенал: `WHERE`, `JOIN`, `GROUP BY`, `ORDER BY`.

```sql
-- только чётные числа из ряда 1..20, по убыванию
SELECT n
FROM generate_series(1, 20) AS g(n)
WHERE n % 2 = 0
ORDER BY n DESC;
```

Её можно **соединять** с настоящими таблицами. Например, `CROSS JOIN` ряда с
категориями строит решётку «число × категория» — так же, как в lab002 мы строили
сетки через `CROSS JOIN` (только там ряд задавался вручную через `VALUES`, а
теперь его порождает функция):

```sql
SELECT g.n, cat.name
FROM generate_series(1, 3) AS g(n)
CROSS JOIN categories AS cat;   -- 3 × (число категорий) строк
```

> **Про алиасы.** Как и у подзапроса в `FROM` (lab004), у SRF в `FROM`
> **желателен алиас** — он даёт имя источнику и его колонкам. Без явного имени
> колонки PostgreSQL назовёт её по имени функции (`generate_series`), что читается
> плохо; поэтому пишите `AS g(n)`.

## Итог файла

- **SRF** — функция, возвращающая **набор строк**, а не одно значение; её место —
  в `FROM`, как у таблицы.
- **`generate_series(start, stop [, step])`** — генератор ряда: числа (шаг по
  умолчанию `1`, может быть отрицательным) или даты/время (шаг — `interval`).
  Оба конца **включаются**; недостижимый `stop` или `NULL` → ноль строк; шаг `0` →
  ошибка. Ряд дат возвращается как метка времени — приводите `::date`.
- Ряд из `generate_series` — это **каркас** (например, сплошной календарь), к
  которому потом приклеивают факты; без него в отчёте были бы дыры.
