# lab001 — Основы SQL

Первая лаба — фундамент. Учимся **доставать** данные из таблицы,
**фильтровать**, **сортировать**, **считать** и **группировать**. Всё это
используется в каждом запросе до конца жизни, поэтому цель — понять не только
синтаксис, но и *как оно работает* и *зачем нужно*.

Теория разбита на короткие документы в папке [`theory/`](theory/) — читайте по
порядку. К каждой теме приложены ссылки на официальную документацию (на
русском). Общая подборка курсов и тренажёров — в [`../RESOURCES.md`](../RESOURCES.md).

## Как запустить

```bash
# из корня репозитория один раз:
docker compose up -d

cd lab001
./run.sh                 # накатит схему и прогонит все запросы

# либо зайти руками и экспериментировать:
../psql.sh lab001
```

## Доменная область

Небольшой интернет-магазин. Три таблицы (полная схема — в [`init.sql`](init.sql)).

### `products` — товары
| Колонка    | Тип            | Смысл                                        |
|------------|----------------|----------------------------------------------|
| `id`       | INT (PK)       | идентификатор товара                          |
| `name`     | TEXT           | название                                      |
| `category` | TEXT           | категория: `electronics`, `books`, `home`, `toys` |
| `price`    | NUMERIC(10,2)  | цена за штуку, ₽                              |
| `in_stock` | INT            | остаток на складе, шт.                        |

### `customers` — клиенты
| Колонка      | Тип      | Смысл             |
|--------------|----------|-------------------|
| `id`         | INT (PK) | идентификатор     |
| `name`       | TEXT     | имя               |
| `city`       | TEXT     | город             |
| `registered` | DATE     | дата регистрации   |

### `orders` — заказы (одна строка = один товар в заказе)
| Колонка       | Тип      | Смысл                   |
|---------------|----------|-------------------------|
| `id`          | INT (PK) | идентификатор строки     |
| `customer_id` | INT (FK) | ссылка на `customers.id` |
| `product_id`  | INT (FK) | ссылка на `products.id`  |
| `quantity`    | INT      | количество               |
| `ordered_at`  | DATE     | дата заказа              |

> В этой лабе задачи решаются в основном на одной таблице `products`;
> `customers`/`orders` даны для контекста. Джойны между таблицами — в `lab002`.
