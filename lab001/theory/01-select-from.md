# 01 — `SELECT` и `FROM`: как читаются данные

[← оглавление лабы](../README.md) · [дальше: WHERE →](02-where.md)

---

## Что это

`SELECT` — команда выборки данных. Это единственный SQL-оператор, который
*читает* и возвращает строки (остальные — `INSERT`, `UPDATE`, `DELETE` —
изменяют данные). Почти вся работа аналитика — это `SELECT`.

Минимальный запрос состоит из двух частей:

```sql
SELECT <что вернуть>
FROM   <откуда взять>;
```

- `FROM` указывает **источник** строк — таблицу (или несколько, или подзапрос).
- `SELECT` указывает **проекцию** — какие колонки/выражения показать.

## Как это выполняется

Важно понять с самого начала: **`FROM` выполняется раньше `SELECT`**. СУБД
сначала берёт строки из таблицы, а уже потом решает, какие колонки показать.
Полный логический порядок исполнения запроса:

```
FROM → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT
```

Из этого следует практическое правило, которое многих сбивает: **в `WHERE`
нельзя ссылаться на алиас из `SELECT`** — на момент выполнения `WHERE` секция
`SELECT` ещё «не случилась». Подробнее про алиасы — ниже.

## Синтаксис и варианты

### Выбор конкретных колонок

```sql
SELECT name, price
FROM products;
```

Вернёт две колонки в указанном порядке. Порядок в `SELECT` = порядок колонок в
результате.

### `*` — все колонки

```sql
SELECT * FROM products;
```

`*` удобно для разведки («что вообще лежит в таблице»), но в рабочих запросах
его избегают:

- при изменении структуры таблицы результат «поедет»;
- тянется лишний трафик и память;
- в джойнах `*` вернёт колонки из всех таблиц, часто с дубликатами имён.

### Квалификация имён (`таблица.колонка`)

Можно (а в джойнах — нужно) указывать, из какой таблицы колонка:

```sql
SELECT products.name, products.price
FROM products;
```

Пока таблица одна — это избыточно, но привыкать к нотации полезно.

### Константы и выражения в `SELECT`

`SELECT` умеет возвращать не только колонки, но и вычисленные значения:

```sql
SELECT
    name,
    price,
    price * 1.20   AS price_with_vat,  -- арифметика
    'товар'        AS kind             -- константа-литерал
FROM products;
```

Подробнее про вычисляемые колонки и `CASE` — в [файле 04](04-case.md).

## Алиасы (`AS`)

Алиас — это временное имя для колонки или таблицы **в пределах одного запроса**.

### Алиас колонки

```sql
SELECT price * in_stock AS stock_value
FROM products;
```

- `AS` можно опустить: `price * in_stock stock_value` — то же самое, но с `AS`
  читается яснее.
- Если алиас содержит пробел или кириллицу — берите в двойные кавычки:
  `AS "Стоимость запаса"`. **Двойные кавычки — это идентификатор** (имя
  колонки), а не строка. Строковые литералы в SQL — в одинарных кавычках
  (`'товар'`).

### Алиас таблицы

```sql
SELECT p.name, p.price
FROM products AS p;   -- дальше к таблице обращаемся как p
```

Сокращает запись и обязателен, когда таблица участвует в запросе дважды
(self-join — будет в lab002).

### Где алиас колонки можно использовать, а где нет

| Секция      | Виден ли алиас из `SELECT`? | Почему |
|-------------|:---------------------------:|--------|
| `WHERE`     | ❌ | выполняется до `SELECT` |
| `GROUP BY`  | ✅ (в PostgreSQL) | расширение стандарта |
| `HAVING`    | ⚠️ частично | лучше повторять выражение |
| `ORDER BY`  | ✅ | выполняется после `SELECT` |

```sql
-- ОШИБКА: алиас stock_value ещё не существует в WHERE
SELECT price * in_stock AS stock_value
FROM products
WHERE stock_value > 100000;      -- ← column "stock_value" does not exist

-- Правильно: повторить выражение в WHERE
SELECT price * in_stock AS stock_value
FROM products
WHERE price * in_stock > 100000;

-- А вот в ORDER BY алиас работает:
SELECT price * in_stock AS stock_value
FROM products
ORDER BY stock_value DESC;
```

## Точка с запятой

`;` завершает оператор. В интерактивном psql без неё запрос не выполнится —
psql будет ждать продолжения. В файлах `;` разделяет несколько операторов.

## Комментарии

```sql
-- однострочный комментарий до конца строки
/* многострочный
   комментарий */
```

## Частые ошибки новичков

- **Строки в одинарных кавычках, идентификаторы — в двойных.**
  `WHERE category = "books"` → ошибка (Postgres ищет колонку `books`).
  Правильно: `WHERE category = 'books'`.
- Забытая `;` в psql — запрос «висит».
- `SELECT *` в проде — см. выше, почему не надо.
