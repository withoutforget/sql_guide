## Что такое агрегат

Агрегатная функция берёт **много строк** и сворачивает их в **одно значение**.
`COUNT`, `SUM`, `AVG`, `MIN`, `MAX` — базовый набор. Без `GROUP BY` агрегат
считается по всей таблице и возвращает **ровно одну строку**.

```sql
SELECT COUNT(*), AVG(price), MAX(price)
FROM products;
```

С `GROUP BY` — по одному значению на каждую группу (см. [файл 06](06-group-having.md)).

## Ключевое правило: агрегаты игнорируют `NULL`

Все агрегаты (кроме `COUNT(*)`) **пропускают строки, где аргумент `NULL`**. Это
не «баг», а определение — но приводит к неожиданностям:

```sql
-- если у части товаров discount IS NULL:
SELECT AVG(discount) FROM products;
-- среднее считается ТОЛЬКО по строкам, где discount задан,
-- а не «сумма / общее число строк». NULL-строки в делителе не участвуют.
```

Если нужно считать `NULL` как ноль — подставьте его явно:
`AVG(COALESCE(discount, 0))`.

## `COUNT` — подсчёт строк. Три разных формы

| Форма                  | Что считает                                        |
|------------------------|----------------------------------------------------|
| `COUNT(*)`             | **все строки** группы, включая с `NULL`            |
| `COUNT(колонка)`       | строки, где `колонка IS NOT NULL`                  |
| `COUNT(DISTINCT колонка)` | число **различных** не-`NULL` значений          |

```sql
SELECT
    COUNT(*)                    AS всего_строк,
    COUNT(discount)             AS с_заполненной_скидкой,
    COUNT(DISTINCT category)    AS сколько_категорий
FROM products;
```

Частая ошибка — ждать от `COUNT(колонка)` того же, что от `COUNT(*)`. Если в
колонке есть `NULL`, числа разойдутся.

## `SUM` — сумма

```sql
SELECT SUM(in_stock) AS всего_единиц FROM products;
```

- Игнорирует `NULL`.
- Если строк нет вообще (или все `NULL`) — вернёт `NULL`, **не 0**. Оборачивайте
  в `COALESCE(SUM(x), 0)`, если нужен ноль.
- Тип результата шире исходного (у `int` → `bigint`), чтобы не переполниться.

## `AVG` — среднее арифметическое

```sql
SELECT ROUND(AVG(price), 2) AS средняя_цена FROM products;
```

- `AVG = SUM / COUNT(колонка)` по не-`NULL` значениям.
- Для целочисленной колонки результат дробный (`numeric`/`double`), деления «в
  целых» не происходит — в отличие от обычного `/` (см. [файл 04](04-case.md)).
- Обычно оборачивают в `ROUND`, чтобы не тянуть длинный хвост.

## `MIN` / `MAX` — минимум и максимум

```sql
SELECT MIN(price) AS дешевле_всех, MAX(price) AS дороже_всех FROM products;
```

- Работают не только с числами: у текста — по алфавиту (collation), у дат — по
  хронологии. `MAX(ordered_at)` — самая свежая дата.
- Игнорируют `NULL`.

## `ROUND` и округление

`ROUND(число, знаков)` — округление до заданного числа знаков после запятой.

```sql
ROUND(10715.4567, 2)   -- 10715.46
ROUND(10715.4567)      -- 10715   (до целого, второй аргумент по умолчанию 0)
```

Родственники: `CEIL(x)` (вверх), `FLOOR(x)` (вниз), `TRUNC(x, n)` (отбросить без
округления). Нюанс: `ROUND(x, n)` со вторым аргументом работает для типа
`numeric`; для `double precision` второго аргумента нет — приведите к `numeric`.

## Условная агрегация: `FILTER` и `CASE`

Часто нужно посчитать агрегат не по всем строкам, а по подмножеству — прямо в
одном запросе, разными «срезами». Два способа:

```sql
-- современный, читаемый — FILTER (PostgreSQL):
SELECT
    COUNT(*)                          AS всего,
    COUNT(*) FILTER (WHERE in_stock = 0)      AS нет_в_наличии,
    AVG(price) FILTER (WHERE category = 'books') AS средняя_по_книгам
FROM products;

-- классический, переносимый — через CASE внутри агрегата:
SELECT
    SUM(CASE WHEN in_stock = 0 THEN 1 ELSE 0 END) AS нет_в_наличии
FROM products;
```

Оба дают один результат; `FILTER` читается лучше, `CASE` работает в любой СУБД.

## `DISTINCT` внутри агрегата

Считать по уникальным значениям умеет любой агрегат:

```sql
SELECT
    COUNT(DISTINCT city)  AS городов,
    COUNT(DISTINCT customer_id) AS активных_клиентов
FROM ...;
```

## Частые ошибки

- **`WHERE COUNT(*) > 1`** — нельзя, агрегаты в `WHERE` запрещены. Фильтр по
  агрегату — это `HAVING` ([файл 06](06-group-having.md)).
- **Смешивать агрегат и обычную колонку без `GROUP BY`**:
  `SELECT category, COUNT(*) FROM products;` → ошибка. Либо всё под агрегат,
  либо `category` в `GROUP BY`.
- Ожидать `0` от `SUM`/`AVG` на пустом наборе — там будет `NULL`.
