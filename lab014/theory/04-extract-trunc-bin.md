# 04 — Извлечение частей и бакетирование: `EXTRACT`, `date_trunc`, `date_bin`

[← текущее время](03-current-time.md) · [оглавление](../README.md) · [дальше: форматирование `to_char` →](05-formatting.md)

---

Две главные операции аналитика над временем: **достать часть** («какой месяц?»,
«какой час?») и **сгруппировать по периоду** («выручка по месяцам», «события по
15-минуткам»). Первое — `EXTRACT`/`date_part`, второе — `date_trunc`/`date_bin`.

## `EXTRACT` — вытащить одно поле

```sql
EXTRACT(поле FROM источник)          -- напр. EXTRACT(month FROM created_at)
date_part('поле', источник)          -- функциональная форма, то же самое
```

Обе возвращают **число** (`numeric` у `EXTRACT`, `double precision` у `date_part`).
`EXTRACT(month FROM ...)` и `date_part('month', ...)` эквивалентны — первая
читается как SQL-стандарт, вторая удобна, когда поле хранится в переменной.

### Поля

| Поле | Что возвращает | Пример (для `2024-03-15 14:30:45`, пятница) |
|------|----------------|---------------------------------------------|
| `year` | год | `2024` |
| `quarter` | квартал 1–4 | `1` |
| `month` | месяц 1–12 | `3` |
| `day` | день месяца 1–31 | `15` |
| `hour` | час 0–23 | `14` |
| `minute` | минута 0–59 | `30` |
| `second` | секунда (с дробной частью) | `45` |
| `dow` | день недели, **0=вс … 6=сб** | `5` (пятница) |
| `isodow` | день недели, **1=пн … 7=вс** | `5` (пятница) |
| `doy` | день года 1–366 | `75` |
| `week` | номер ISO-недели 1–53 | `11` |
| `epoch` | секунд с 1970-01-01 00:00 UTC (для интервала — всего секунд) | `1710513045` |

> ⚠️ **`dow` против `isodow` — источник ошибок «выходных».** У `dow` неделя
> начинается с воскресенья и оно = 0, суббота = 6. У `isodow` неделя начинается с
> понедельника (=1), воскресенье = 7. Для «суббота или воскресенье» удобнее
> **`isodow IN (6,7)`** — оба выходных стоят рядом в конце. Через `dow` это было бы
> `dow IN (0,6)` — легко ошибиться. В задачах 5 и 15 используем `isodow`.

### `EXTRACT(epoch FROM ...)` — в секунды

Особое, очень полезное поле:

- для **момента** (`timestamptz`) — число секунд с «эпохи Unix»
  (1970-01-01 00:00 UTC). `EXTRACT(epoch FROM timestamptz '1970-01-01 00:00+00')` = `0`;
- для **интервала** — **вся длительность в секундах**. Это главный способ перевести
  «человеческую» разницу времён в число:

```sql
SELECT EXTRACT(epoch FROM interval '1 day 1 hour');   -- 90000  (24·3600 + 3600)
SELECT EXTRACT(epoch FROM (delivered_at - created_at)) / 3600 AS часов;
--   разницу двух timestamptz (interval) переводим в часы
```

Так считают длительности «в числах»: секунды/`60` → минуты, `/3600` → часы,
`/86400` → дни (см. задачу 6).

## `date_trunc` — «обрезать» до начала периода

```sql
date_trunc('единица', источник)
```

Обнуляет все поля **мельче** указанной единицы — получаем начало периода, к
которому принадлежит момент. Это основа группировки временных рядов «по
месяцам/неделям/часам»: все моменты одного месяца после `date_trunc('month', …)`
становятся **одинаковыми** (первым днём месяца) и попадают в одну группу.

Единицы: `microseconds`, `milliseconds`, `second`, `minute`, `hour`, `day`,
`week`, `month`, `quarter`, `year`, `decade`, `century`, `millennium`.

```sql
-- для 2024-03-15 14:30:45 :
date_trunc('hour',    ts)  -- 2024-03-15 14:00:00
date_trunc('day',     ts)  -- 2024-03-15 00:00:00
date_trunc('week',    ts)  -- 2024-03-11 00:00:00   ← понедельник!
date_trunc('month',   ts)  -- 2024-03-01 00:00:00
date_trunc('quarter', ts)  -- 2024-01-01 00:00:00
date_trunc('year',    ts)  -- 2024-01-01 00:00:00
```

Тип результата совпадает с входом (`date_trunc` над `timestamptz` даёт
`timestamptz`, над `timestamp` — `timestamp`). Есть форма с третьим аргументом-зоной:
`date_trunc('day', ts, 'Europe/Moscow')` — усечение по «локальному» дню (то же, что
через `AT TIME ZONE`, см. ниже).

> ⚠️ **Неделя начинается с понедельника.** `date_trunc('week', …)` всегда даёт
> понедельник (по ISO). Если бизнес считает неделю с воскресенья — `date_trunc` «в
> лоб» не подойдёт, придётся сдвигать (`date_trunc('week', ts + interval '1 day') -
> interval '1 day'`). Помните об этом при недельных отчётах.

### Группировка по месяцам — типовой приём

```sql
SELECT date_trunc('month', created_at) AS месяц,
       sum(amount)                     AS выручка
FROM orders
GROUP BY 1
ORDER BY 1;
```

Все заказы месяца сворачиваются в одну строку с ключом «первое число месяца».
Часто ключ дополнительно форматируют в `'YYYY-MM'` через `to_char` (файл 05) — но
**группировать** удобнее по `date_trunc` (это дата, она правильно сортируется и
ложится на индекс), а `to_char` применять только для красивого вывода.

### Учёт часового пояса при усечении

`date_trunc('day', created_at)` для `timestamptz` режет по границе дня **в поясе
сессии**. Чтобы сгруппировать по «локальному дню» конкретного региона независимо от
сессии — сначала переведите момент в настенное время этой зоны:

```sql
date_trunc('day', created_at AT TIME ZONE 'Europe/Moscow')
--   момент 2024-03-15 23:30 UTC → в Москве это 16 марта 02:30 → день = 2024-03-16
```

## `date_bin` — бакеты произвольного размера (PostgreSQL 14+)

`date_trunc` умеет только «стандартные» периоды. Когда нужны корзины **своей**
ширины (по 15 минут, по 10 дней, по 6 часов) — берут `date_bin`:

```sql
date_bin(ширина_интервал, источник, начало_отсчёта)
```

Раскладывает ось времени на равные интервалы `ширина`, начиная от `начало_отсчёта`
(origin), и возвращает **левую границу** корзины, в которую попал момент:

```sql
SELECT date_bin(interval '15 min',
                 timestamptz '2024-03-15 14:37:00+00',
                 timestamptz '2024-03-15 00:00:00+00');
-- 2024-03-15 14:30:00+00   (14:37 попадает в корзину [14:30, 14:45))

SELECT date_bin(interval '2 hour', ts, timestamptz '2024-03-15 00:00:00+00');
-- 2024-03-15 14:00:00+00   (корзина [14:00, 16:00))
```

Корзины **полуоткрытые** `[начало, начало+ширина)`: правая граница принадлежит уже
следующей корзине. `начало_отсчёта` задаёт «фазу» сетки — обычно берут полночь
нужного дня. Ограничения: `ширина` должна быть положительной и не содержать
месяцев/лет (только фиксированные единицы — минуты/часы/дни/недели), потому что
месяц не фиксированной длины. Для помесячной группировки используйте `date_trunc`.

В задаче 12 группируем события по 15-минутным корзинам через `date_bin`.
